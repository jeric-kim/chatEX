# 톡톡 (chatEX)

모바일 웹에서 카카오톡처럼 대화 목록과 실시간 대화 화면을 체험할 수 있는 싱글 페이지 예제입니다. GitHub Pages에 바로 올릴 수 있도록 순수 HTML/CSS/JS로 구성되어 있습니다.

## 주요 기능
- **로그인**: 일회용 닉네임(미입력 시 랜덤)과 비밀번호로 로그인합니다. 로그아웃 전까지 로그인 정보가 유지되며, 로그아웃 시 모든 정보가 삭제됩니다.
- **채팅방 목록**: 최신/오래된 순으로 정렬하고, 수동 새로고침/자동 새로고침(5, 10, 15초)를 설정할 수 있습니다. 목록은 최초 로그인 시 비어 있으며 새 채팅 생성 후 최신 순으로 정렬됩니다.
- **새 채팅 시작**: 우측 하단 **+** 버튼으로 닉네임을 검색하고, 검색 버튼 또는 Enter로 필터링한 뒤 선택하여 대화를 시작합니다. 내장된 다양한 닉네임 추천 목록을 검색할 수 있으며, 결과가 없으면 입력한 닉네임으로 바로 새 대화를 만들 수 있습니다.
- **공용 WebSocket 실시간 대화**: 브라우저/기기/네트워크가 달라도, 같은 페이지에 로그인하면 서로 닉네임을 검색해 바로 메시지를 교환합니다. `demo.piesocket.com` 공용 WebSocket 채널을 통해 대화/목록이 실시간 동기화되고, 각 클라이언트는 받은 데이터를 로컬에 캐시해 새 탭에서도 이어 볼 수 있습니다.
- **실제 대화 화면**: 채팅방을 터치하면 메시지 화면으로 이동하며, 상단 뒤로 가기로 목록으로 복귀할 수 있습니다. 내가 보낸 말풍선/상대 말풍선을 구분해 보여주며, 상대 답변이 도착하면 NEW 배지가 표시됩니다.
- **메시지 정보**: 내용은 15자까지 목록에 표시 후 말줄임, 보낸 일시는 `YYYY-MM-DD HH:MM:SS` 형태로 출력합니다.
- **모바일 친화 UI**: 작은 화면에서도 편하게 쓸 수 있도록 레이아웃과 버튼 크기를 조정했습니다.

## 빠르게 실행하기
1. 저장소를 클론하거나 ZIP으로 내려받습니다.
2. `index.html`을 브라우저에서 직접 열면 바로 체험할 수 있습니다.
3. GitHub Pages에서 사용하려면 이 저장소를 원격 저장소로 푸시한 뒤, Pages 소스를 `main` 브랜치(루트)로 지정하세요.

## 사용법
- 로그인 폼에서 닉네임(선택)과 비밀번호를 입력하고 로그인합니다.
- 우측 하단 **+** 버튼을 눌러 닉네임을 검색하고 새 채팅을 만듭니다. 검색 결과가 없을 때는 입력한 닉네임으로 바로 시작할 수 있습니다.
- 다른 탭/기기에서 동일 페이지를 열어 다른 닉네임으로 로그인하면, 서로를 검색해서 선택 후 **채팅 시작하기**를 누르면 양쪽 모두 실시간으로 메시지가 동기화됩니다(공용 WebSocket 사용). 네트워크가 끊겼을 땐 로컬 캐시로 보이며, 연결이 복구되면 자동으로 맞춰집니다.
- 공용 채널인 만큼 민감한 정보는 입력하지 마세요. 연결이 되지 않으면 네트워크를 확인하거나 잠시 후 다시 시도하면 자동으로 재연결됩니다.
- 채팅방을 선택하면 대화 화면으로 이동하며, 메시지를 보내면 상대 응답이 도착합니다. 목록의 NEW 배지는 채팅을 열면 해제됩니다.
- 로그아웃 버튼을 누르면 저장된 세션과 채팅 목록이 모두 지워집니다.

## 배포 팁 (GitHub Pages)
1. GitHub 저장소를 만든 뒤 이 코드를 푸시합니다.
2. 저장소 **Settings > Pages**에서 **Source**를 `Deploy from a branch`, **Branch**를 `main / (root)`로 선택합니다.
3. 저장 후 제공되는 GitHub Pages 주소로 접속하면 서비스가 바로 동작합니다.
